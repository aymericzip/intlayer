# آپ کو Intlayer پر کیوں غور کرنا چاہئے؟

## Intlayer کیا ہے؟

**Intlayer** ایک بین الاقوامی سازی (internationalization) لائبریری ہے جو خاص طور پر جاوا اسکرپٹ ڈویلپرز کے لیے ڈیزائن کی گئی ہے۔ یہ آپ کے کوڈ میں ہر جگہ اپنے مواد کا اعلان کرنے کی اجازت دیتی ہے۔ یہ کثیر لسانی مواد کے اعلانات کو آپ کے کوڈ میں آسانی سے ضم کرنے کے لیے منظم لغات میں تبدیل کرتی ہے۔ ٹائپ اسکرپٹ (TypeScript) کا استعمال کرتے ہوئے، **Intlayer** آپ کی ترقی کو مضبوط اور موثر بناتی ہے۔

## Intlayer کیوں بنایا گیا؟

Intlayer کو ایک عام مسئلے کو حل کرنے کے لیے بنایا گیا تھا جو تمام عام i18n لائبریریوں جیسے `next-intl` ، `react-i18next` ، `react-intl` ، `next-i18next` ، `react-intl` ، اور `vue-i18n` کو متاثر کرتا ہے۔

یہ تمام حل آپ کے مواد کو درج کرنے اور منظم کرنے کے لیے ایک مرکزی نقطہ نظر اپناتے ہیں۔ مثال کے طور پر:

```bash
.
├── locales
│   ├── en.json
│   ├── es.json
│   └── fr.json
├── i18n.ts
└── src
    └── components
        └── MyComponent
            └── index.tsx
```

یا یہاں نیم اسپیس (namespaces) کا استعمال کرتے ہوئے:

```bash
.
├── locales
│   ├── en
│   │  ├── footer.json
│   │  └── navbar.json
│   ├── fr
│   │  ├── footer.json
│   │  └── navbar.json
│   └── es
│      ├── footer.json
│      └── navbar.json
├── i18n.ts
└── src
    └── components
        └── MyComponent
            └── index.tsx
```

اس قسم کا فن تعمیر ترقی کے عمل کو سست کر دیتا ہے اور کئی وجوہات کی بنا پر کوڈ بیس کو برقرار رکھنا زیادہ پیچیدہ بنا دیتا ہے:

1. **کسی بھی نئے بنائے گئے جزو (component) کے لیے، آپ کو یہ کرنا چاہیے:**
   - `locales` فولڈر میں نیا وسیلہ/نیم اسپیس بنائیں
   - اپنے پیج میں نیا نیم اسپیس درآمد کرنا یاد رکھیں
   - اپنے مواد کا ترجمہ کریں (اکثر AI فراہم کنندگان سے کاپی/پیسٹ کر کے دستی طور پر کیا جاتا ہے)

2. **اپنے اجزاء میں کی جانے والی کسی بھی تبدیلی کے لیے، آپ کو یہ کرنا چاہیے:**
   - متعلقہ وسیلہ/نیم اسپیس تلاش کریں (جزو سے دور)
   - اپنے مواد کا ترجمہ کریں
   - یقینی بنائیں کہ آپ کا مواد کسی بھی زبان کے لیے اپ ٹو ڈیٹ ہے
   - تصدیق کریں کہ آپ کے نیم اسپیس میں غیر استعمال شدہ کلیدیں/قدریں شامل نہیں ہیں
   - یقینی بنائیں کہ آپ کی JSON فائلوں کی ساخت تمام زبانوں کے لیے ایک جیسی ہے

ان حلوں کا استعمال کرنے والے پیشہ ورانہ پروجیکٹس پر، اکثر لوکلائزیشن پلیٹ فارمز آپ کے مواد کے ترجمے کو منظم کرنے میں مدد کے لیے استعمال کیے جاتے ہیں۔ تاہم، بڑے پروجیکٹس کے لیے یہ تیزی سے مہنگا ہو سکتا ہے۔

اس مسئلے کو حل کرنے کے لیے، Intlayer ایک ایسا نقطہ نظر اپناتا ہے جو آپ کے مواد کو فی جزو (per-component) اسکوپ کرتا ہے اور آپ کے مواد کو آپ کے جزو کے قریب رکھتا ہے، جیسا کہ ہم اکثر CSS (`styled-components`) ، اقسام (types) ، دستاویزات (`storybook`) ، یا یونٹ ٹیسٹ (`jest`) کے ساتھ کرتے ہیں۔

```bash codeFormat="typescript"
.
└── components
    └── MyComponent
        ├── index.content.ts
        ├── index.test.tsx
        ├── index.stories.tsx
        └── index.tsx
```

```bash codeFormat="commonjs"
.
└── components
    └── MyComponent
        ├── index.content.cjs
        ├── index.test.mjs
        ├── index.stories.mjs
        └── index.tsx
```

```bash codeFormat="esm"
.
└── components
    └── MyComponent
        ├── index.content.mjs
        ├── index.test.mjs
        ├── index.stories.mjs
        └── index.tsx
```

```tsx fileName="./components/MyComponent/index.content.ts" codeFormat={["typescript", "esm", "commonjs"]}
import { t, type Dictionary } from "intlayer";

const componentExampleContent = {
  key: "component-example",
  content: {
    myTranslatedContent: t({
      en: "Hello World",
      es: "Hola Mundo",
      fr: "Bonjour le monde",
    }),
  },
} satisfies Dictionary;

export default componentExampleContent;
```

```tsx fileName="./components/MyComponent/index.tsx" codeFormat={["typescript", "esm"]}
import { useIntlayer } from "react-intlayer";

export const ComponentExample = () => {
  const { myTranslatedContent } = useIntlayer("component-example");

  return <span>{myTranslatedContent}</span>;
};
```

یہ نقطہ نظر آپ کو درج ذیل کی اجازت دیتا ہے:

1. **ترقی کی رفتار بڑھائیں**
   - `.content.{{ts|mjs|cjs|json}}` فائلیں VSCode ایکسٹینشن کا استعمال کرتے ہوئے بنائی جا سکتی ہیں
   - آپ کے IDE میں آٹو مکمل AI ٹولز (جیسے GitHub Copilot) آپ کو اپنے مواد کا اعلان کرنے میں مدد کر سکتے ہیں، جس سے کاپی/پیسٹ کم ہو جاتا ہے

2. **اپنے کوڈ بیس کو صاف کریں**
   - پیچیدگی کو کم کریں
   - دیکھ بھال کی اہلیت کو بڑھائیں

3. **اپنے اجزاء اور ان سے متعلقہ مواد کو زیادہ آسانی سے نقل کریں (مثال: لاگ ان/رجسٹر جزو، وغیرہ)**
   - دوسرے اجزاء کے مواد کو متاثر کرنے کے خطرے کو محدود کر کے
   - بیرونی انحصار کے بغیر اپنے مواد کو ایک ایپلیکیشن سے دوسری ایپلیکیشن میں کاپی/پیسٹ کر کے

4. **غیر استعمال شدہ اجزاء کے لیے غیر استعمال شدہ کلیدوں/قدروں کے ساتھ اپنے کوڈ بیس کو آلودہ کرنے سے بچیں**
   - اگر آپ کوئی جزو استعمال نہیں کرتے ہیں، تو Intlayer اس سے متعلقہ مواد درآمد نہیں کرے گا
   - اگر آپ کسی جزو کو حذف کرتے ہیں، تو آپ کو اس سے متعلقہ مواد کو حذف کرنا زیادہ آسانی سے یاد رہے گا کیونکہ وہ اسی فولڈر میں موجود ہو گا

5. **اپنے کثیر لسانی مواد کا اعلان کرنے کے لیے AI ایجنٹوں کے لیے استدلال کی لاگت کو کم کریں**
   - AI ایجنٹ کو یہ جاننے کے لیے آپ کے پورے کوڈ بیس کو اسکین نہیں کرنا پڑے گا کہ آپ کا مواد کہاں لاگو کرنا ہے
   - ترجمہ آپ کے IDE میں آٹو مکمل AI ٹولز (جیسے GitHub Copilot) کے ذریعے آسانی سے کیا جا سکتا ہے

6. **لوڈنگ کی کارکردگی کو بہتر بنائیں**
   - اگر کوئی جزو سست لوڈ (lazy-loaded) ہے، تو اس سے متعلقہ مواد اسی وقت لوڈ ہو گا

## انٹرآپریبلٹی

`intlayer` آپ کے `react-intl` ، `react-i18next` ، `next-intl` ، `next-i18next` ، اور `vue-i18n` نیم اسپیس کو سنبھالنے میں بھی مدد کر سکتا ہے۔

`intlayer` کا استعمال کرتے ہوئے، آپ اپنی پسندیدہ i18n لائبریری کے فارمیٹ میں اپنے مواد کا اعلان کر سکتے ہیں، اور intlayer آپ کی پسند کی جگہ پر آپ کے نیم اسپیس تیار کرے گا (مثال: `/messages/{{locale}}/{{namespace}}.json`)۔
