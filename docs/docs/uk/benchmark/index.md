# Benchmark Bloom — Звіт

## Поточні результати

Відвідайте [**інтерактивну панель бенчмарку**](https://intlayer.org/benchmark) для отримання живого порівняння та узагальнених даних.
| `scoped-dynamic` | Висока (витік майже нульовий) | Висока |

Перехід від `static` до `scoped-dynamic` зазвичай зменшує кількість невикористаного контенту на 60–90%, але вимагає значно більше налаштувань. Бібліотеки на кшталт Intlayer автоматизують паттерн scoped-dynamic, тож розробники отримують ефективність без зайвого повторюваного коду (boilerplate).

### Як розуміти цифри витоку (leakage)

Витік сторінки **35%** означає, що 35% завантаженого JavaScript для цієї сторінки містить рядки з інших сторінок — контент, який користувач не може бачити на цій сторінці. На сторінці розміром 400 КБ це ~140 КБ зайвих даних.

Витік локалі (locale leakage) **10%** означає, що 10% бандла містить переклади мовами, якими поточний користувач не користується.

### Реактивність проти часу рендерингу

- **E2E реактивність** вимірює повний досвід користувача: мережа, накладні витрати фреймворку, оновлення DOM.
- **Час React Profiler** ізолює вартість повторного рендерингу дерева React.

Бібліотека може мати низький час Profiler, але високий час E2E, якщо перемикання локалі передбачає мережевий запит (отримання нового файлу локалі). Навпаки, бібліотека може мати високий час Profiler, але все одно відчуватися швидкою, якщо вона ефективно групує оновлення.
