# Nieuwe Intlayer v9 - Wat is er nieuw?

Welkom bij Intlayer v9! Deze grote release markeert een enorme mijlpaal in het vereenvoudigen van het migratiepad naar Intlayer met **Compat-adapterpakketten** voor grote i18n-bibliotheken (`react-i18next`, `next-intl`, `vue-i18n`, enz.) en voegt ondersteuning toe voor rijke inhoudsstructuren: **Verzamelingen** en **Varianten**.

## Compat-adapterpakketten

Migreren naar Intlayer vanuit populaire i18n-bibliotheken is nu eenvoudiger dan ooit. We hebben vijf compatibiliteitspakketten gemaakt die exact dezelfde publieke API's blootstellen als standaard i18n-bibliotheken, maar al het vertaalwerk tijdens uitvoering delegeren aan Intlayer.

### Dezelfde publieke API blootstellen met strikte typering

Door imports te vervangen, profiteert u van alle voordelen van Intlayer (inclusief compilatietijd type-veiligheid ten opzichte van uw werkelijke woordenboeken) met minimale codewijzigingen:

- `@intlayer/i18next`
- `@intlayer/react-i18next`
- `@intlayer/next-intl`
- `@intlayer/react-intl`
- `@intlayer/next-i18next`
- `@intlayer/vue-i18n`
- `@intlayer/lingui`

Wijzig bijvoorbeeld eenvoudig:

```ts
import { useTranslation } from "react-i18next";
```

naar:

```ts
import { useTranslation } from "@intlayer/react-i18next";
```

Uw sleutels worden nu **strikt getypeerd** op basis van uw Intlayer-woordenboeken, wat volledige dot-pad-autoaanvulling biedt in uw IDE!

### Bundler-aliasplugins (Vite, Next.js, Turbopack)

Om te migreren zonder alle importstatements handmatig te herschrijven, bevat elk compat-adapterpakket een **aangepaste bundler-plugin** (Vite of Next.js) onder het `/plugin`-subpad.

Deze plugins herschrijven automatisch bestaande imports (bijv. `react-i18next` of `next-intl`) naar hun `@intlayer/*`-equivalenten tijdens het bouwen.

#### Next.js (Webpack / Turbopack) Voorbeeld

Gebruik in plaats van `withIntlayer` de compat-plugin om uw Next.js-configuratie te wikkelen:

```ts fileName="next.config.ts"
import { createNextI18nPlugin } from "@intlayer/next-i18next/plugin";
import type { NextConfig } from "next";

const withIntlayer = createNextI18nPlugin();
const nextConfig: NextConfig = {};

export default withIntlayer(nextConfig);
```

#### Vite (React, Vue, Solid, Svelte) Voorbeeld

```ts fileName="vite.config.ts"
import vueI18nVitePlugin from "@intlayer/vue-i18n/plugin";

export default defineConfig({
  plugins: [vueI18nVitePlugin()],
});
```

## Gedeelde runtime-resolver

Alle compat-adapters routeren vertaalresolutie nu via één sterk geoptimaliseerde runtime-parser: `@intlayer/core/messageFormat`.

- **Berichtinterpolatie**: Verwerkt standaard `{{var}}` (witruimte & dot-paden), ICU-geformatteerde argumenten (`{v, number, percent}` enz.) en gewone `{var}`-sjablonen.
- **Berichtknooppuntresolver**: Verwerkt geneste knooppunten: `insert()`, `plural()` (CLDR meervoudsregels), `enu()` (opsomming), `gender()`, HTML-tags, arrays en aanroepbare functieknooppunten.
- **Getokeniseerde tagparser**: Ondersteunt inline XML/HTML-tags en genummerde tags (bijv. `<1>children</1>`) voor rijke-tekstweergave zonder extra configuratie.

## Functiespecificaties: Verzamelingen en varianten

Intlayer v9 gaat verder dan statische sleutel-waardeparen en stelt woordenboeken in staat dynamische layoutstructuren te declareren die end-to-end volledig getypeerd zijn.

### 1. Verzamelingen

Definieer een door CMS beheerde lijst met geordende items (bijv. FAQ's, producten of bloglijsten). Elk item bevindt zich in een eigen `.content.ts`-bestand met dezelfde `key` en een `item`-nummer:

```ts fileName="faq.1.content.ts"
import { t, type Dictionary } from "intlayer";

export default {
  key: "faq",
  item: 1,
  content: {
    question: t({ en: "What is Intlayer?", fr: "Qu'est-ce qu'Intlayer ?" }),
    answer: t({ en: "An i18n toolkit.", fr: "Une boîte à outils i18n." }),
  },
} satisfies Dictionary;
```

```ts fileName="faq.2.content.ts"
import { t, type Dictionary } from "intlayer";

export default {
  key: "faq",
  item: 2,
  content: {
    question: t({ en: "Is it free?", fr: "Est-ce gratuit ?" }),
    answer: t({ en: "Yes, open-source.", fr: "Oui, open-source." }),
  },
} satisfies Dictionary;
```

#### Gebruik:

```ts
// Alle items als array ophalen
const allFaqs = useIntlayer("faq"); // -> { question: string, answer: string }[]

// Eén item ophalen op index
const faq = useIntlayer("faq", { item: 2 }); // -> { question: string, answer: string }
```

### 2. Varianten

Lever A/B-tests, seizoensgebonden koppen, functievlaggen of aangepaste landingspagina's:

#### Tekstvarianten (bijv. A/B-testen)

```ts fileName="hero.content.ts"
import { t, type Dictionary } from "intlayer";

export default {
  key: "hero-banner",
  variant: "default",
  content: {
    control: t({ en: "Welcome", fr: "Bienvenue" }),
    black_friday: t({ en: "Shop now", fr: "Acheter maintenant" }),
  },
} satisfies Dictionary;
```

#### Gebruik:

```ts
const banner = useIntlayer("hero-banner", { variant: "black_friday" });
```

#### Objectvarianten (bijv. Dynamische records)

```ts fileName="product.content.ts"
import { t, type Dictionary } from "intlayer";

export default {
  key: "product-copy",
  variant: {
    id: "prod_123",
    category: "books",
  },
  content: {
    title: t({ en: "Clean Code", fr: "Code Propre" }),
  },
} satisfies Dictionary;
```

#### Gebruik:

```ts
// Haalt alleen het gevraagde record dynamisch op (vereist Suspense)
const product = useIntlayer("product-copy", {
  variant: { id: "prod_123", category: "books" },
});
```

## Vite-plugin: ingebouwde compiler en proxy

De `intlayer()` Vite-plugin bundelt nu de **compiler** en de **locale-routeringsproxy** rechtstreeks, zodat de meeste projecten slechts één plugin nodig hebben in `vite.config.ts`:

```ts fileName="vite.config.ts"
import { defineConfig } from "vite";
import { intlayer } from "vite-intlayer";

export default defineConfig({
  plugins: [intlayer()],
});
```

- **Compiler**: Wordt automatisch geactiveerd wanneer `compiler.enabled` is ingesteld op `true` en een `compiler.output`-pad is geconfigureerd. U hoeft `intlayerCompiler()` niet langer afzonderlijk te registreren.
- **Proxy**: Wordt automatisch geactiveerd op basis van de nieuwe optie `routing.enableProxy` (standaard `true`). Het koppelt de middleware voor taaldetedering, omleiding en herschrijving in ontwikkeling, voorvertoning en productie-SSR. U hoeft `intlayerProxy()` niet langer afzonderlijk te registreren.

### De optie `routing.enableProxy`

Een nieuwe optie `routing.enableProxy` bepaalt of de locale-routeringsproxy wordt gekoppeld. De standaardwaarde is `true`. Schakel deze uit wanneer u de taalroutering zelf wilt afhandelen:

```ts fileName="intlayer.config.ts"
import type { IntlayerConfig } from "intlayer";

const config: IntlayerConfig = {
  routing: {
    enableProxy: false, // Standaard: true
  },
};

export default config;
```

De zelfstandige plugins `intlayerCompiler()` en `intlayerProxy()` blijven beschikbaar voor geavanceerde installaties. Het is veilig om ze samen met `intlayer()` te registreren — elke plugin dedupliceert zichzelf en wordt slechts één keer uitgevoerd.

## Compiler standaard uitgeschakeld

Vanaf Intlayer v9 is de **compiler standaard uitgeschakeld** (`compiler.enabled` is nu standaard `false`). Om build-tijdextractie van uw `.content.ts`-bestanden in te schakelen, stelt u `compiler.enabled: true` in uw configuratie in:

```ts fileName="intlayer.config.ts"
import type { IntlayerConfig } from "intlayer";

const config: IntlayerConfig = {
  compiler: {
    enabled: true, // Standaard: false — vereist om de compiler in te schakelen sinds v9
    output: ({ fileName }) => `./${fileName}.content.ts`,
  },
};

export default config;
```

Wanneer de compiler is uitgeschakeld, slaat Intlayer de build-tijdextractiestap over en vertrouwt op uw reeds gedeclareerde woordenboeken. Schakel deze alleen in wanneer u wilt dat de bundler-plugin (`@intlayer/swc`, `@intlayer/babel` of de `intlayer()` Vite-plugin) inhoud automatisch extraheert.

## React Native: importeren vanuit één pakket

In een React Native- of Expo-app hoeft u niet langer zowel `react-intlayer` als `react-native-intlayer` te beheren. Het pakket `react-native-intlayer` **herexporteert nu de volledige `react-intlayer`-API** (hooks, hulpprogramma's en de subpaden `/format`, `/html` en `/markdown`), en de `IntlayerProvider` past de React Native polyfills automatisch toe.

Importeer alles vanuit het enkele pakket `react-native-intlayer`:

```tsx
import {
  IntlayerProvider,
  useIntlayer,
  useLocale,
} from "react-native-intlayer";
```

```bash
npm install intlayer react-native-intlayer
```

> Importeren vanuit `react-intlayer` blijft werken, maar `react-native-intlayer` is nu het aanbevolen enkelvoudige invoerpunt voor React Native — de provider bevat de polyfills die de webgerichte `react-intlayer`-provider niet heeft.

## CMS SDK: gebruik Intlayer als headless-contentdatabase

Intlayer v9 levert een overzichtelijke, automatisch authenticerende SDK in `@intlayer/api` om programmatisch met de CMS te communiceren — haal projecten op, haal woordenboeken op en duw of update ze vanuit uw eigen server, scripts of CI. Authenticatie (OAuth2 `client_credentials`) wordt voor u beheerd en vernieuwd.

De SDK is opgesplitst in **twee afzonderlijke imports** zodat uw bundel alleen de domeinen bevat die u daadwerkelijk gebruikt:

1. `createIntlayerCMS` — een lichtgewicht **authenticator** die de referenties en het beheerde token bewaart (geen domein-client gebundeld).
2. `dictionaryEndpoint`, `projectEndpoint`, … — per-domein **endpoint-binders**, elk geïmporteerd vanuit hun eigen subpad.

```ts fileName="cms.ts"
import { createIntlayerCMS } from "@intlayer/api";
import { dictionaryEndpoint } from "@intlayer/api/dictionary";

const cms = createIntlayerCMS();

// Lezen
const { data: dictionaries } = await dictionaryEndpoint(cms).getDictionaries();

// Schrijven — gebruik de CMS als een database
await dictionaryEndpoint(cms).pushDictionaries([myDictionary]);
```

> Beveiliging: de CMS-referenties verlenen schrijftoegang tot uw inhoud. Maak de authenticator altijd alleen aan **aan de serverzijde** — stuur `clientId`/`clientSecret` nooit naar de browser.

## Zelf hosten

Intlayer v9 biedt eersteklas ondersteuning voor het uitvoeren van uw eigen Intlayer-instantie met één opdracht — geen Intlayer Cloud-account vereist.

```sh
curl -fsSL https://intlayer.org/install.sh | sh
```

Het installatieprogramma downloadt een `docker-compose.yml` en een `.env`, genereert automatisch de vereiste geheimen en voert `docker compose up -d` uit. Alles — het dashboard, de API, de database, objectopslag en transactionele e-mail — wordt lokaal in containers uitgevoerd.

## Migratienotities vanuit v8

Als u upgradet vanuit v8, let dan op dat v9 geen breaking changes bevat. Maar dit zijn de belangrijkste wijzigingen:

- **Compiler standaard uitgeschakeld**: `compiler.enabled` is nu standaard `false`. Als u afhankelijk bent van build-tijdextractie van uw `.content.ts`-bestanden, stelt u `compiler.enabled: true` in uw `intlayer.config.ts` in.
- **Vite-plugin**: De compiler en locale-routeringsproxy zijn nu gebundeld in `intlayer()`. Als u eerder `intlayerCompiler()` of `intlayerProxy()` handmatig registreerde, kunt u ze verwijderen — ze worden automatisch gededupliceerd, dus ze bewaren is onschadelijk. Gebruik `routing.enableProxy: false` om de proxy uit te schakelen.
- **Talen en dialecten**: Als u externe i18n-afhankelijkheden gebruikt, voeg dan de bijbehorende compat-adapterplugins toe in uw configuratie of bundler-installatie om imports automatisch te herschrijven.
- **Aangepaste selectors**: Bij het aanroepen van `useIntlayer` is de tweede parameter nu gereserveerd voor een optieobject met `{ locale, item, variant }`. Als u eerder een taalstring direct doorgaf, kunt u dat nog steeds doen, maar het gebruik van het optieobject wordt aanbevolen voor geavanceerde selecties.
- **React Native**: `react-native-intlayer` herexporteert nu de volledige `react-intlayer`-API. Importeer in een React Native-app alles vanuit `react-native-intlayer` en verwijder de directe `react-intlayer`-afhankelijkheid. Bestaande `react-intlayer`-imports blijven werken.
