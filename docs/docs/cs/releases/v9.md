# Nový Intlayer v9 - Co je nového?

Vítejte v Intlayer v9! Toto hlavní vydání představuje obrovský milník ve zjednodušení migrace na Intlayer díky **balíčkům compat adaptérů** pro hlavní i18n knihovny (`react-i18next`, `next-intl`, `vue-i18n` atd.) a přidává podporu pro bohaté struktury obsahu: **Kolekce** a **Varianty**.

## Balíčky compat adaptérů

Migrace na Intlayer z oblíbených i18n knihoven je nyní snazší než kdykoli předtím. Vytvořili jsme pět balíčků kompatibility, které vystavují **přesně stejná veřejná API** jako standardní i18n knihovny, ale při běhu delegují veškerou překladovou práci na Intlayer.

### Vystavení stejného veřejného API se striktním typováním

Nahrazením importů získáte všechny výhody Intlayeru (včetně typové bezpečnosti v době kompilace vůči vašim skutečným slovníkům) s minimálními změnami kódu:

- `@intlayer/i18next`
- `@intlayer/react-i18next`
- `@intlayer/next-intl`
- `@intlayer/react-intl`
- `@intlayer/next-i18next`
- `@intlayer/vue-i18n`
- `@intlayer/lingui`

Jednoduše například změňte:

```ts
import { useTranslation } from "react-i18next";
```

na:

```ts
import { useTranslation } from "@intlayer/react-i18next";
```

Vaše klíče budou nyní **striktně typovány** vůči vašim Intlayer slovníkům a nabídnou plné automatické doplňování tečkových cest ve vašem IDE!

### Pluginy pro aliasy bundlerů (Vite, Next.js, Turbopack)

Aby bylo možné migrovat bez ručního přepisování všech importů, každý balíček compat adaptéru obsahuje **vlastní plugin pro bundler** (Vite nebo Next.js) pod subcestou `/plugin`.

Tyto pluginy automaticky přepisují stávající importy (např. `react-i18next` nebo `next-intl`) na jejich ekvivalenty `@intlayer/*` v době sestavení.

#### Příklad Next.js (Webpack / Turbopack)

Místo `withIntlayer` zabalte konfiguraci Next.js pomocí pluginu kompatibility:

```ts fileName="next.config.ts"
import { createNextI18nPlugin } from "@intlayer/next-i18next/plugin";
import type { NextConfig } from "next";

const withIntlayer = createNextI18nPlugin();
const nextConfig: NextConfig = {};

export default withIntlayer(nextConfig);
```

#### Příklad Vite (React, Vue, Solid, Svelte)

```ts fileName="vite.config.ts"
import vueI18nVitePlugin from "@intlayer/vue-i18n/plugin";

export default defineConfig({
  plugins: [vueI18nVitePlugin()],
});
```

## Sdílený runtime resolver

Všechny compat adaptéry nyní směrují rozlišení překladů přes jeden vysoce optimalizovaný runtime parser: `@intlayer/core/messageFormat`.

- **Interpolace zpráv**: Zpracovává standardní `{{var}}` (bílé znaky a tečkové cesty), argumenty ve formátu ICU (`{v, number, percent}` atd.) a holé šablony `{var}`.
- **Resolver uzlů zpráv**: Zpracovává vnořené uzly: `insert()`, `plural()` (pravidla CLDR pro množné číslo), `enu()` (výčet), `gender()`, HTML tagy, pole a volatelné funkční uzly.
- **Tokenizovaný parser tagů**: Podporuje vložené XML/HTML tagy a číslované tagy (např. `<1>children</1>`) pro vykreslování formátovaného textu přímo z krabice.

## Specifikace funkcí: Kolekce a varianty

Intlayer v9 jde za hranice statických objektů klíč–hodnota a umožňuje slovníkům deklarovat dynamické rozložení struktur, které jsou plně typovány end-to-end.

### 1. Kolekce

Definujte seznam seřazených položek spravovaných CMS (např. FAQ, produkty nebo seznamy blogů). Každá položka žije ve vlastním souboru `.content.ts` se stejným `key` a číslem `item`:

```ts fileName="faq.1.content.ts"
import { t, type Dictionary } from "intlayer";

export default {
  key: "faq",
  item: 1,
  content: {
    question: t({ en: "What is Intlayer?", fr: "Qu'est-ce qu'Intlayer ?" }),
    answer: t({ en: "An i18n toolkit.", fr: "Une boîte à outils i18n." }),
  },
} satisfies Dictionary;
```

```ts fileName="faq.2.content.ts"
import { t, type Dictionary } from "intlayer";

export default {
  key: "faq",
  item: 2,
  content: {
    question: t({ en: "Is it free?", fr: "Est-ce gratuit ?" }),
    answer: t({ en: "Yes, open-source.", fr: "Oui, open-source." }),
  },
} satisfies Dictionary;
```

#### Použití:

```ts
// Načíst všechny položky jako pole
const allFaqs = useIntlayer("faq"); // -> { question: string, answer: string }[]

// Načíst jednu položku podle indexu
const faq = useIntlayer("faq", { item: 2 }); // -> { question: string, answer: string }
```

### 2. Varianty

Poskytujte A/B testy, sezónní záhlaví, příznaky funkcí nebo vlastní vstupní stránky:

#### Textové varianty (např. A/B testování)

```ts fileName="hero.content.ts"
import { t, type Dictionary } from "intlayer";

export default {
  key: "hero-banner",
  variant: "default",
  content: {
    control: t({ en: "Welcome", fr: "Bienvenue" }),
    black_friday: t({ en: "Shop now", fr: "Acheter maintenant" }),
  },
} satisfies Dictionary;
```

#### Použití:

```ts
const banner = useIntlayer("hero-banner", { variant: "black_friday" });
```

#### Objektové varianty (např. Dynamické záznamy)

```ts fileName="product.content.ts"
import { t, type Dictionary } from "intlayer";

export default {
  key: "product-copy",
  variant: {
    id: "prod_123",
    category: "books",
  },
  content: {
    title: t({ en: "Clean Code", fr: "Code Propre" }),
  },
} satisfies Dictionary;
```

#### Použití:

```ts
// Načte pouze požadovaný záznam dynamicky (vyžaduje Suspense)
const product = useIntlayer("product-copy", {
  variant: { id: "prod_123", category: "books" },
});
```

## Dynamické načítání a optimalizace velikosti bundlu

Aby byly bundly extrémně malé, Intlayer v9 podporuje dynamické líné načítání.

Ve své konfiguraci nastavte `importMode` na `'dynamic'` nebo `'fetch'`:

```ts fileName="intlayer.config.ts"
export default {
  dictionary: {
    importMode: "dynamic", // "static" | "dynamic" | "fetch"
  },
};
```

V době sestavení `@intlayer/swc` a `@intlayer/babel` prohledají vaše soubory a nahradí volání `useIntlayer`/`getIntlayer` wrappers tree-shakeable (`useDictionary`/`useDictionaryDynamic`). Načítá se pouze obsah potřebný pro vybranou položku kolekce, variantu nebo jazyk, čímž se zabrání tomu, aby váš produkční bundle obsahoval nevyužité překlady.

## Vite plugin: zabudovaný kompilátor a proxy

Plugin Vite `intlayer()` nyní přímo sdružuje **kompilátor** a **proxy pro směrování jazyků**, takže většina projektů potřebuje pouze jeden plugin v `vite.config.ts`:

```ts fileName="vite.config.ts"
import { defineConfig } from "vite";
import { intlayer } from "vite-intlayer";

export default defineConfig({
  plugins: [intlayer()],
});
```

- **Kompilátor**: Aktivuje se automaticky, když je `compiler.enabled` nastaveno na `true` a je nakonfigurována cesta `compiler.output`. `intlayerCompiler()` již není třeba registrovat samostatně.
- **Proxy**: Aktivuje se automaticky na základě nové možnosti `routing.enableProxy` (výchozí hodnota `true`). Propojuje middleware pro detekci jazyka, přesměrování a přepisování ve vývoji, náhledu a produkčním SSR. `intlayerProxy()` již není třeba registrovat samostatně.

### Možnost `routing.enableProxy`

Nová možnost `routing.enableProxy` řídí, zda je proxy pro směrování jazyků zapojena. Výchozí hodnota je `true`. Vypněte ji, když chcete směrování jazyků zpracovávat sami:

```ts fileName="intlayer.config.ts"
import type { IntlayerConfig } from "intlayer";

const config: IntlayerConfig = {
  routing: {
    enableProxy: false, // Výchozí: true
  },
};

export default config;
```

Samostatné pluginy `intlayerCompiler()` a `intlayerProxy()` zůstávají exportovány pro pokročilá nastavení. Jejich registrace společně s `intlayer()` je bezpečná — každý plugin se deduplikuje a spustí se pouze jednou.

## Kompilátor je ve výchozím nastavení zakázán

Počínaje Intlayer v9 je **kompilátor ve výchozím nastavení zakázán** (`compiler.enabled` nyní výchozí hodnota je `false`). Chcete-li zapnout extrakci vašich souborů `.content.ts` v době sestavení, nastavte `compiler.enabled: true` ve své konfiguraci:

```ts fileName="intlayer.config.ts"
import type { IntlayerConfig } from "intlayer";

const config: IntlayerConfig = {
  compiler: {
    enabled: true, // Výchozí: false — vyžadováno pro povolení kompilátoru od v9
    output: ({ fileName }) => `./${fileName}.content.ts`,
  },
};

export default config;
```

Když je kompilátor zakázán, Intlayer přeskočí krok extrakce v době sestavení a spoléhá na vaše již deklarované slovníky. Povolte jej pouze tehdy, když chcete, aby plugin bundleru (`@intlayer/swc`, `@intlayer/babel` nebo plugin Vite `intlayer()`) automaticky extrahoval obsah.

## React Native: importy z jednoho balíčku

V aplikaci React Native nebo Expo již nemusíte žonglovat s `react-intlayer` i `react-native-intlayer`. Balíček `react-native-intlayer` nyní **znovu exportuje celé API `react-intlayer`** (hooks, utility a subcesty `/format`, `/html` a `/markdown`) a jeho `IntlayerProvider` automaticky aplikuje polyfilly React Native.

Importujte vše z jediného balíčku `react-native-intlayer`:

```tsx
import {
  IntlayerProvider,
  useIntlayer,
  useLocale,
} from "react-native-intlayer";
```

```bash
npm install intlayer react-native-intlayer
```

> Import z `react-intlayer` nadále funguje, ale `react-native-intlayer` je nyní doporučeným jediným vstupním bodem pro React Native — jeho provider dodává polyfilly, které webově orientovaný provider `react-intlayer` nemá.

## CMS SDK: použijte Intlayer jako headless databázi obsahu

Intlayer v9 dodává přehledné, automaticky autentizující SDK v `@intlayer/api` pro programatickou interakci s CMS — načítejte projekty, načítejte slovníky a odesílejte nebo aktualizujte je z vlastního serveru, skriptů nebo CI. Autentizace (OAuth2 `client_credentials`) je spravována a obnovována za vás.

SDK je rozděleno do **dvou samostatných importů**, aby váš bundle obsahoval pouze domény, které skutečně používáte:

1. `createIntlayerCMS` — lehký **autentizátor** uchovávající přihlašovací údaje a spravovaný token (žádný klientský bundle domény není součástí).
2. `dictionaryEndpoint`, `projectEndpoint`, … — **binery endpointů** pro jednotlivé domény, každý importovaný z vlastní subcesty.

```ts fileName="cms.ts"
import { createIntlayerCMS } from "@intlayer/api";
import { dictionaryEndpoint } from "@intlayer/api/dictionary";

const cms = createIntlayerCMS();

// Čtení
const { data: dictionaries } = await dictionaryEndpoint(cms).getDictionaries();

// Zápis — použijte CMS jako databázi
await dictionaryEndpoint(cms).pushDictionaries([myDictionary]);
```

> Zabezpečení: přihlašovací údaje CMS udělují oprávnění k zápisu vašeho obsahu. Autentizátor vytvářejte vždy pouze **na straně serveru** — nikdy neodesílejte `clientId`/`clientSecret` do prohlížeče.

## Vlastní hostování

Intlayer v9 poskytuje prvotřídní podporu pro provozování vlastní instance Intlayeru jediným příkazem — bez nutnosti účtu Intlayer Cloud.

```sh
curl -fsSL https://intlayer.org/install.sh | sh
```

Instalátor stáhne `docker-compose.yml` a `.env`, automaticky vygeneruje požadovaná tajemství a spustí `docker compose up -d`. Vše — dashboard, API, databáze, objektové úložiště a transakční e-mail — běží lokálně v kontejnerech.

## Poznámky k migraci z v8

Pokud provádíte upgrade z v8, vezměte na vědomí, že v9 neobsahuje žádné zlomové změny. Zde jsou však klíčové změny:

- **Kompilátor ve výchozím nastavení zakázán**: `compiler.enabled` nyní výchozí hodnota je `false`. Pokud se spoléháte na extrakci souborů `.content.ts` v době sestavení, nastavte `compiler.enabled: true` v `intlayer.config.ts`.
- **Plugin Vite**: Kompilátor a proxy pro směrování jazyků jsou nyní součástí `intlayer()`. Pokud jste dříve ručně registrovali `intlayerCompiler()` nebo `intlayerProxy()`, můžete je odstranit — jsou automaticky deduplikovány, takže jejich ponechání je neškodné. Použijte `routing.enableProxy: false` pro vypnutí proxy.
- **Jazyky a dialekty**: Pokud používáte externí závislosti i18n, přidejte příslušné compat adaptérové pluginy do vaší konfigurace nebo nastavení bundleru pro automatické přepisování importů.
- **Vlastní selektory**: Při volání `useIntlayer` je druhý parametr nyní vyhrazen pro objekt možností obsahující `{ locale, item, variant }`. Pokud jste dříve předávali řetězec jazyka přímo, stále to lze, ale použití objektu možností je doporučeno pro pokročilé výběry.
- **React Native**: `react-native-intlayer` nyní znovu exportuje celé API `react-intlayer`. V aplikaci React Native importujte vše z `react-native-intlayer` a odeberte přímou závislost na `react-intlayer`. Stávající importy z `react-intlayer` nadále fungují.
