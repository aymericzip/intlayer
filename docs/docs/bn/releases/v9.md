# নতুন Intlayer v9 - কী নতুন?

Intlayer v9-এ স্বাগতম! এই প্রধান রিলিজটি প্রধান i18n লাইব্রেরিগুলির (`react-i18next`, `next-intl`, `vue-i18n` ইত্যাদি) জন্য **Compat Adapter Packages** সহ Intlayer-এ মাইগ্রেশনের পথ সহজ করার ক্ষেত্রে একটি বিশাল মাইলফলক চিহ্নিত করে এবং সমৃদ্ধ কন্টেন্ট কাঠামোর সমর্থন যোগ করে: **Collections** এবং **Variants**।

## Compat Adapter Packages

জনপ্রিয় i18n লাইব্রেরি থেকে Intlayer-এ মাইগ্রেট করা এখন আগের চেয়ে অনেক সহজ। আমরা পাঁচটি compat প্যাকেজ তৈরি করেছি যা স্ট্যান্ডার্ড i18n লাইব্রেরির **হুবহু একই পাবলিক API** প্রকাশ করে কিন্তু রানটাইমে সমস্ত অনুবাদ কাজ Intlayer-এ অর্পণ করে।

### একই পাবলিক API কঠোর টাইপিং সহ প্রকাশ করুন

ইম্পোর্ট প্রতিস্থাপন করে, আপনি ন্যূনতম কোড পরিবর্তনের সাথে Intlayer-এর সমস্ত সুবিধা পান (আপনার প্রকৃত ডিকশনারিগুলির বিপরীতে কম্পাইল-টাইম টাইপ নিরাপত্তা সহ):

- `@intlayer/i18next`
- `@intlayer/react-i18next`
- `@intlayer/next-intl`
- `@intlayer/react-intl`
- `@intlayer/next-i18next`
- `@intlayer/vue-i18n`
- `@intlayer/lingui`

উদাহরণস্বরূপ, শুধু পরিবর্তন করুন:

```ts
import { useTranslation } from "react-i18next";
```

থেকে:

```ts
import { useTranslation } from "@intlayer/react-i18next";
```

আপনার কীগুলি এখন আপনার Intlayer ডিকশনারিগুলির বিপরীতে **কঠোরভাবে টাইপ করা** হবে, আপনার IDE-তে সম্পূর্ণ ডট-পাথ অটো-কমপ্লিশন অফার করবে!

### Bundler Alias Plugins (Vite, Next.js, Turbopack)

সমস্ত ইম্পোর্ট স্টেটমেন্ট ম্যানুয়ালি পুনরায় লেখা ছাড়াই মাইগ্রেট করার অনুমতি দিতে, প্রতিটি compat adapter প্যাকেজ `/plugin` সাবপাথের অধীনে একটি **কাস্টম bundler plugin** (Vite অথবা Next.js) অন্তর্ভুক্ত করে।

এই প্লাগিনগুলি স্বয়ংক্রিয়ভাবে বিল্ড সময়ে বিদ্যমান ইম্পোর্টগুলি (যেমন `react-i18next` বা `next-intl`) তাদের `@intlayer/*` সমতুল্যে পুনরায় লেখে।

#### Next.js (Webpack / Turbopack) উদাহরণ

`withIntlayer`-এর পরিবর্তে, আপনার Next.js কনফিগারেশনকে compat প্লাগিন দিয়ে মোড়ান:

```ts fileName="next.config.ts"
import { createNextI18nPlugin } from "@intlayer/next-i18next/plugin";
import type { NextConfig } from "next";

const withIntlayer = createNextI18nPlugin();
const nextConfig: NextConfig = {};

export default withIntlayer(nextConfig);
```

#### Vite (React, Vue, Solid, Svelte) উদাহরণ

```ts fileName="vite.config.ts"
import vueI18nVitePlugin from "@intlayer/vue-i18n/plugin";

export default defineConfig({
  plugins: [vueI18nVitePlugin()],
});
```

## Mutualized Runtime Resolver

সমস্ত compat adapter এখন একটি একক, উচ্চ-অপ্টিমাইজড রানটাইম পার্সারের মাধ্যমে অনুবাদ রেজোলিউশন রুট করে: `@intlayer/core/messageFormat`।

- **Interpolate Message**: স্ট্যান্ডার্ড `{{var}}` (হোয়াইটস্পেস ও ডট-পাথ), ICU ফরম্যাটেড আর্গুমেন্ট (`{v, number, percent}` ইত্যাদি), এবং সরল `{var}` টেমপ্লেট রিজলভ করে।
- **Message Node Resolver**: নেস্টেড নোড রিজলভ করে: `insert()`, `plural()` (CLDR plural নিয়ম), `enu()` (গণনা), `gender()`, HTML ট্যাগ, অ্যারে, এবং কল করযোগ্য ফাংশন নোড।
- **Tokenized Tag Parser**: ইনলাইন XML/HTML ট্যাগ এবং সংখ্যাযুক্ত ট্যাগ (যেমন, `<1>children</1>`) সমর্থন করে রিচ-টেক্সট রেন্ডারিং বাক্সের বাইরে পরিচালনা করতে।

## ফিচার স্পেক: Collections ও Variants

Intlayer v9 স্ট্যাটিক কী-ভ্যালু অবজেক্টের বাইরে প্রসারিত হয়, ডিকশনারিগুলিকে ডায়নামিক লেআউট কাঠামো ঘোষণা করার অনুমতি দেয় যা শেষ থেকে শেষ পর্যন্ত সম্পূর্ণ টাইপ করা।

### ১। Collections

CMS-পরিচালিত ক্রমানুসারে আইটেমের তালিকা সংজ্ঞায়িত করুন (যেমন FAQ, পণ্য, বা ব্লগ তালিকা)। প্রতিটি আইটেম তার নিজস্ব `.content.ts` ফাইলে একই `key` এবং একটি `item` নম্বর সহ থাকে:

```ts fileName="faq.1.content.ts"
import { t, type Dictionary } from "intlayer";

export default {
  key: "faq",
  item: 1,
  content: {
    question: t({ en: "What is Intlayer?", fr: "Qu'est-ce qu'Intlayer ?" }),
    answer: t({ en: "An i18n toolkit.", fr: "Une boîte à outils i18n." }),
  },
} satisfies Dictionary;
```

```ts fileName="faq.2.content.ts"
import { t, type Dictionary } from "intlayer";

export default {
  key: "faq",
  item: 2,
  content: {
    question: t({ en: "Is it free?", fr: "Est-ce gratuit ?" }),
    answer: t({ en: "Yes, open-source.", fr: "Oui, open-source." }),
  },
} satisfies Dictionary;
```

#### ব্যবহার:

```ts
// সমস্ত আইটেম অ্যারে হিসেবে আনুন
const allFaqs = useIntlayer("faq"); // -> { question: string, answer: string }[]

// ইন্ডেক্স দ্বারা একটি একক আইটেম আনুন
const faq = useIntlayer("faq", { item: 2 }); // -> { question: string, answer: string }
```

### ২। Variants

A/B পরীক্ষা, মৌসুমী হেডার, ফিচার ফ্ল্যাগ, বা কাস্টম ল্যান্ডিং পেজ সরবরাহ করুন:

#### স্ট্রিং ভেরিয়েন্ট (যেমন A/B পরীক্ষা)

```ts fileName="hero.content.ts"
import { t, type Dictionary } from "intlayer";

export default {
  key: "hero-banner",
  variant: "default",
  content: {
    control: t({ en: "Welcome", fr: "Bienvenue" }),
    black_friday: t({ en: "Shop now", fr: "Acheter maintenant" }),
  },
} satisfies Dictionary;
```

#### ব্যবহার:

```ts
const banner = useIntlayer("hero-banner", { variant: "black_friday" });
```

#### অবজেক্ট ভেরিয়েন্ট (যেমন ডায়নামিক রেকর্ড)

```ts fileName="product.content.ts"
import { t, type Dictionary } from "intlayer";

export default {
  key: "product-copy",
  variant: {
    id: "prod_123",
    category: "books",
  },
  content: {
    title: t({ en: "Clean Code", fr: "Code Propre" }),
  },
} satisfies Dictionary;
```

#### ব্যবহার:

```ts
// শুধুমাত্র অনুরোধ করা রেকর্ড ডায়নামিকভাবে আনে (Suspense প্রয়োজন)
const product = useIntlayer("product-copy", {
  variant: { id: "prod_123", category: "books" },
});
```

## ডায়নামিক লোডিং ও বান্ডেল সাইজ অপ্টিমাইজেশন

বান্ডেলগুলি অত্যন্ত ছোট রাখতে, Intlayer v9 ডায়নামিক লেজি লোডিং সমর্থন করে।

আপনার কনফিগারেশনে, `importMode` কে `'dynamic'` বা `'fetch'`-এ সেট করুন:

```ts fileName="intlayer.config.ts"
export default {
  dictionary: {
    importMode: "dynamic", // "static" | "dynamic" | "fetch"
  },
};
```

বিল্ড সময়ে, `@intlayer/swc` এবং `@intlayer/babel` আপনার ফাইলগুলি স্ক্যান করে এবং `useIntlayer` / `getIntlayer` কলগুলিকে ট্রি-শেকেবল র‍্যাপার দিয়ে প্রতিস্থাপন করে (`useDictionary` / `useDictionaryDynamic`)। শুধুমাত্র নির্বাচিত collection আইটেম, ভেরিয়েন্ট, বা লোকেলের জন্য প্রয়োজনীয় কন্টেন্ট লোড করা হয়, আপনার প্রোডাকশন বান্ডেলকে অব্যবহৃত অনুবাদ ধারণ করা থেকে রক্ষা করে।

## Vite প্লাগিন: বান্ডেলড কম্পাইলার ও প্রক্সি

`intlayer()` Vite প্লাগিন এখন সরাসরি **কম্পাইলার** এবং **লোকেল-রাউটিং প্রক্সি** বান্ডেল করে, তাই বেশিরভাগ প্রজেক্টের শুধুমাত্র `vite.config.ts`-এ একটি একক প্লাগিন প্রয়োজন:

```ts fileName="vite.config.ts"
import { defineConfig } from "vite";
import { intlayer } from "vite-intlayer";

export default defineConfig({
  plugins: [intlayer()],
});
```

- **কম্পাইলার**: স্বয়ংক্রিয়ভাবে সক্রিয় হয় যখন `compiler.enabled` কে `true`-এ সেট করা হয় এবং একটি `compiler.output` পাথ কনফিগার করা থাকে। আপনাকে আর আলাদাভাবে `intlayerCompiler()` নিবন্ধন করতে হবে না।
- **প্রক্সি**: নতুন `routing.enableProxy` বিকল্পের উপর ভিত্তি করে স্বয়ংক্রিয়ভাবে সক্রিয় হয় (ডিফল্টরূপে `true`)। এটি ডেভেলপমেন্ট, প্রিভিউ, এবং প্রোডাকশন SSR-এ লোকেল ডিটেকশন / রিডাইরেক্ট / রিরাইট মিডলওয়্যার সংযুক্ত করে। আপনাকে আর আলাদাভাবে `intlayerProxy()` নিবন্ধন করতে হবে না।

### `routing.enableProxy` বিকল্প

একটি নতুন `routing.enableProxy` বিকল্প নিয়ন্ত্রণ করে লোকেল-রাউটিং প্রক্সি প্লাগ ইন করা হয়েছে কিনা। এটি ডিফল্টরূপে `true`। নিজে লোকেল রাউটিং পরিচালনা করতে চাইলে এটি অক্ষম করুন:

```ts fileName="intlayer.config.ts"
import type { IntlayerConfig } from "intlayer";

const config: IntlayerConfig = {
  routing: {
    enableProxy: false, // ডিফল্ট: true
  },
};

export default config;
```

স্বতন্ত্র `intlayerCompiler()` এবং `intlayerProxy()` প্লাগিনগুলি উন্নত সেটআপের জন্য এক্সপোর্ট করা থাকে। এগুলি `intlayer()`-এর সাথে নিবন্ধন করা নিরাপদ — প্রতিটি প্লাগিন নিজেকে ডিডুপ্লিকেট করে এবং শুধুমাত্র একবার চলে।

## কম্পাইলার ডিফল্টরূপে অক্ষম

Intlayer v9 থেকে শুরু করে, **কম্পাইলার ডিফল্টরূপে অক্ষম** (`compiler.enabled` এখন ডিফল্টরূপে `false`)। আপনার `.content.ts` ফাইলগুলির বিল্ড-টাইম এক্সট্রাকশনে অপ্ট ইন করতে, আপনার কনফিগারেশনে `compiler.enabled: true` সেট করুন:

```ts fileName="intlayer.config.ts"
import type { IntlayerConfig } from "intlayer";

const config: IntlayerConfig = {
  compiler: {
    enabled: true, // ডিফল্ট: false — v9 থেকে কম্পাইলার সক্রিয় করতে প্রয়োজন
    output: ({ fileName }) => `./${fileName}.content.ts`,
  },
};

export default config;
```

কম্পাইলার অক্ষম থাকলে, Intlayer বিল্ড-টাইম এক্সট্রাকশন ধাপ এড়িয়ে যায় এবং আপনার ইতিমধ্যে ঘোষিত ডিকশনারিগুলির উপর নির্ভর করে। এটি শুধুমাত্র তখনই সক্ষম করুন যখন আপনি চান bundler প্লাগিন (`@intlayer/swc`, `@intlayer/babel`, বা `intlayer()` Vite প্লাগিন) স্বয়ংক্রিয়ভাবে কন্টেন্ট এক্সট্র্যাক্ট করুক।

## React Native: একক-প্যাকেজ ইম্পোর্ট

একটি React Native বা Expo অ্যাপে, আপনাকে আর `react-intlayer` এবং `react-native-intlayer` উভয়ই ম্যানেজ করতে হবে না। `react-native-intlayer` প্যাকেজ এখন **সম্পূর্ণ `react-intlayer` API** পুনরায় এক্সপোর্ট করে (হুক, ইউটিলিটি, এবং `/format`, `/html`, এবং `/markdown` সাবপাথ), এবং এর `IntlayerProvider` স্বয়ংক্রিয়ভাবে React Native পলিফিল প্রয়োগ করে।

একক `react-native-intlayer` প্যাকেজ থেকে সবকিছু ইম্পোর্ট করুন:

```tsx
import {
  IntlayerProvider,
  useIntlayer,
  useLocale,
} from "react-native-intlayer";
```

```bash
npm install intlayer react-native-intlayer
```

> `react-intlayer` থেকে ইম্পোর্ট করা কাজ করতে থাকে, কিন্তু `react-native-intlayer` এখন React Native-এর জন্য প্রস্তাবিত একক এন্ট্রি পয়েন্ট — এর প্রোভাইডারে পলিফিল শিপ করা হয় যা ওয়েব-ভিত্তিক `react-intlayer` প্রোভাইডারে নেই।

## CMS SDK: Intlayer কে একটি হেডলেস কন্টেন্ট ডেটাবেস হিসেবে ব্যবহার করুন

Intlayer v9 `@intlayer/api`-এ একটি পরিষ্কার, স্বয়ং-প্রমাণীকরণকারী SDK শিপ করে CMS-এর সাথে প্রোগ্রামাটিক্যালি ইন্টারঅ্যাক্ট করতে — প্রজেক্ট আনুন, ডিকশনারি আনুন, এবং আপনার নিজের সার্ভার, স্ক্রিপ্ট, বা CI থেকে সেগুলি পুশ বা আপডেট করুন। প্রমাণীকরণ (OAuth2 `client_credentials`) আপনার জন্য পরিচালিত এবং রিফ্রেশ করা হয়।

SDK **দুটি আলাদা ইম্পোর্টে** বিভক্ত যাতে আপনার বান্ডেলে শুধুমাত্র আপনার ব্যবহৃত ডোমেইনগুলি অন্তর্ভুক্ত হয়:

১। `createIntlayerCMS` — একটি হালকা **অথেনটিকেটর** যা ক্রেডেনশিয়াল এবং পরিচালিত টোকেন ধারণ করে (কোনো ডোমেইন ক্লায়েন্ট বান্ডেল করা নেই)।
২। `dictionaryEndpoint`, `projectEndpoint`, … — প্রতি-ডোমেইন **এন্ডপয়েন্ট বাইন্ডার**, প্রতিটি তার নিজস্ব সাবপাথ থেকে ইম্পোর্ট করা হয়।

```ts fileName="cms.ts"
import { createIntlayerCMS } from "@intlayer/api";
import { dictionaryEndpoint } from "@intlayer/api/dictionary";

const cms = createIntlayerCMS();

// পড়ুন
const { data: dictionaries } = await dictionaryEndpoint(cms).getDictionaries();

// লিখুন — CMS কে ডেটাবেস হিসেবে ব্যবহার করুন
await dictionaryEndpoint(cms).pushDictionaries([myDictionary]);
```

> নিরাপত্তা: CMS ক্রেডেনশিয়াল আপনার কন্টেন্টে লেখার অ্যাক্সেস প্রদান করে। অথেনটিকেটর সর্বদা **সার্ভার সাইডে** তৈরি করুন — কখনই `clientId` / `clientSecret` ব্রাউজারে পাঠাবেন না।

## সেলফ-হোস্টিং

Intlayer v9 একটি একক কমান্ড দিয়ে আপনার নিজস্ব Intlayer ইনস্ট্যান্স চালানোর জন্য প্রথম-শ্রেণির সমর্থন শিপ করে — কোনো Intlayer Cloud অ্যাকাউন্ট প্রয়োজন নেই।

```sh
curl -fsSL https://intlayer.org/install.sh | sh
```

ইন্সটলার একটি `docker-compose.yml` এবং একটি `.env` ডাউনলোড করে, প্রয়োজনীয় সিক্রেটগুলি স্বয়ংক্রিয়ভাবে তৈরি করে, এবং `docker compose up -d` চালায়। সমস্ত কিছু — ড্যাশবোর্ড, API, ডেটাবেস, অবজেক্ট স্টোরেজ, এবং ট্রানজ্যাকশনাল ইমেইল — কন্টেইনারে স্থানীয়ভাবে চলে।

## v8 থেকে মাইগ্রেশন নোট

যদি আপনি v8 থেকে আপগ্রেড করছেন, মনে রাখবেন যে v9-এ কোনো ব্রেকিং পরিবর্তন নেই। তবে এখানে মূল পরিবর্তনগুলি রয়েছে:

- **কম্পাইলার ডিফল্টরূপে অক্ষম**: `compiler.enabled` এখন ডিফল্টরূপে `false`। যদি আপনি আপনার `.content.ts` ফাইলগুলির বিল্ড-টাইম এক্সট্রাকশনের উপর নির্ভর করেন, তাহলে আপনার `intlayer.config.ts`-এ `compiler.enabled: true` সেট করুন।
- **Vite প্লাগিন**: কম্পাইলার এবং লোকেল-রাউটিং প্রক্সি এখন `intlayer()`-এ বান্ডেল করা হয়েছে। আপনি যদি আগে ম্যানুয়ালি `intlayerCompiler()` বা `intlayerProxy()` নিবন্ধন করে থাকেন, তাহলে সেগুলি সরাতে পারেন — সেগুলি স্বয়ংক্রিয়ভাবে ডিডুপ্লিকেট হয়, তাই রাখা নিরাপদ। প্রক্সি থেকে অপ্ট আউট করতে `routing.enableProxy: false` ব্যবহার করুন।
- **Locales ও Dialects**: যদি বাহ্যিক i18n নির্ভরতা ব্যবহার করেন, তাহলে ইম্পোর্ট স্বয়ংক্রিয়ভাবে পুনরায় লিখতে আপনার কনফিগারেশন বা bundler সেটআপে সংশ্লিষ্ট compat adapter প্লাগিন যোগ করুন।
- **কাস্টম সিলেক্টর**: `useIntlayer` কল করার সময়, দ্বিতীয় প্যারামিটার এখন `{ locale, item, variant }` ধারণকারী একটি বিকল্প অবজেক্টের জন্য সংরক্ষিত। আপনি যদি আগে সরাসরি একটি লোকেল স্ট্রিং পাস করে থাকেন, তাহলে এখনও করতে পারেন, কিন্তু উন্নত নির্বাচনের জন্য অপশন অবজেক্ট ব্যবহার করার পরামর্শ দেওয়া হয়।
- **React Native**: `react-native-intlayer` এখন সম্পূর্ণ `react-intlayer` API পুনরায় এক্সপোর্ট করে। একটি React Native অ্যাপে, `react-native-intlayer` থেকে সবকিছু ইম্পোর্ট করুন এবং সরাসরি `react-intlayer` নির্ভরতা বাদ দিন। বিদ্যমান `react-intlayer` ইম্পোর্ট কাজ করতে থাকে।
