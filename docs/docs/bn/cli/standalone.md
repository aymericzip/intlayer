# স্ট্যান্ডঅলোন বান্ডল (Standalone Bundle)

`standalone` কমান্ডটি আপনাকে Intlayer এবং অন্য যেকোনো নির্দিষ্ট প্যাকেজ সম্বলিত একটি স্ট্যান্ডঅলোন JavaScript বান্ডল তৈরি করার অনুমতি দেয়। এটি প্যাকেজ ম্যানেজার বা বান্ডলার নেই এমন পরিবেশে Intlayer ব্যবহার করার জন্য বিশেষভাবে দরকারী, যেমন একটি সাধারণ HTML/JS অ্যাপ।

বান্ডলটি অনুরোধ করা প্যাকেজ এবং তাদের নির্ভরতাগুলোকে একটি একক ফাইলে একত্রিত করতে [esbuild](https://esbuild.github.io/) ব্যবহার করে যা যেকোনো ওয়েব প্রজেক্টে সহজেই ইম্পোর্ট করা যায়।

## ব্যবহার

```bash
npx intlayer standalone --packages [প্যাকেজ...] [বিকল্প]
```

## বিকল্প

- `-o, --outfile [outfile]` - ঐচ্ছিক। আউটপুট ফাইলের নাম। ডিফল্ট: `intlayer-bundle.js`।
- `--packages [প্যাকেজ...]` - প্রয়োজনীয়। বান্ডলে অন্তর্ভুক্ত করার জন্য প্যাকেজগুলোর তালিকা (যেমন: `intlayer`, `vanilla-intlayer`)।
- `--version [version]` - ঐচ্ছিক। বান্ডল করার জন্য প্যাকেজগুলোর সংস্করণ। নির্দিষ্ট না করা থাকলে, ডিফল্টভাবে Intlayer CLI-এর সংস্করণ ব্যবহার করা হয়।
- `--minify` - ঐচ্ছিক। আউটপুটটি ছোট (minify) করা হবে কি না। ডিফল্ট: `true`।
- `--platform [platform]` - ঐচ্ছিক। বান্ডলের জন্য টার্গেট প্ল্যাটফর্ম (যেমন: `browser`, `node`)। ডিফল্ট: `browser`।
- `--format [format]` - ঐচ্ছিক। বান্ডলের জন্য আউটপুট ফরম্যাট (যেমন: `esm`, `cjs`, `iife`)। ডিফল্ট: `esm`।

## সাধারণ বিকল্প

- `--env-file [envFile]` - পরিবেশ ফাইল।
- `-e, --env [env]` - পরিবেশ।
- `--base-dir [baseDir]` - বেস ডিরেক্টরি।
- `--no-cache` - ক্যাশে নিষ্ক্রিয় করুন।
- `--verbose` - বিস্তারিত আউটপুট দেখান।

## উদাহরণ:

### Vanilla JS-এর জন্য একটি বান্ডল তৈরি করা:

```bash
npx intlayer standalone --packages intlayer vanilla-intlayer --outfile intlayer.js
```

এটি `intlayer` এবং `vanilla-intlayer` উভয় প্যাকেজ সম্বলিত একটি `intlayer.js` ফাইল তৈরি করবে, যা ছোট করা হবে এবং ESM ফরম্যাটে হবে, এবং একটি `<script>` ট্যাগের মাধ্যমে ব্রাউজারে ব্যবহারের জন্য প্রস্তুত থাকবে।

### নির্দিষ্ট সংস্করণ বান্ডল করা:

```bash
npx intlayer standalone --packages intlayer --version 8.6.4
```

### ভিন্ন ফরম্যাটে বান্ডল করা:

```bash
npx intlayer standalone --packages intlayer --format iife
```

## এটি কী করে:

১. **একটি অস্থায়ী পরিবেশ তৈরি করে** - নির্ভরতাগুলো পরিচালনা করার জন্য একটি অস্থায়ী ডিরেক্টরি সেট আপ করে।
২. **প্যাকেজ ইনস্টল করে** - অনুরোধ করা প্যাকেজ এবং তাদের নির্ভরতাগুলো ইনস্টল করার জন্য `npm` বা `bun` (যদি উপলব্ধ থাকে) ব্যবহার করে।
৩. **এন্ট্রি পয়েন্ট তৈরি করে** - একটি অস্থায়ী এন্ট্রি পয়েন্ট ফাইল তৈরি করে যা সমস্ত অনুরোধ করা প্যাকেজগুলোকে এক্সপোর্ট করে এবং ব্রাউজারে চালানোর সময় সেগুলোকে গ্লোবাল ভেরিয়েবল হিসেবে প্রকাশ করে।
৪. **esbuild দিয়ে বান্ডল করে** - অনুরোধ অনুযায়ী মিনিফিকেশন এবং ফরম্যাটিং প্রয়োগ করে সবকিছুকে একটি একক ফাইলে একত্রিত করতে esbuild ব্যবহার করে।
৫. **ফাইল তৈরি করে** - ফলাফলস্বরূপ বান্ডলটিকে নির্দিষ্ট আউটপুট পাথে লেখে।

## গ্লোবাল ভেরিয়েবল

যখন বান্ডলটি ব্রাউজারে লোড হয়, তখন এটি অনুরোধ করা প্যাকেজগুলোকে `window` অবজেক্টে গ্লোবাল ভেরিয়েবল হিসেবে প্রকাশ করে। ভেরিয়েবলের নামগুলো প্যাকেজের নাম থেকে নেওয়া হয় (যেমন: `intlayer` হয়ে যায় `Intlayer`, এবং `vanilla-intlayer` হয়ে যায় `VanillaIntlayer`)।

```javascript
// বান্ডল থেকে Intlayer অ্যাক্সেস করা
const { getLocaleName } = window.Intlayer;
const { installIntlayer, useIntlayer } = window.VanillaIntlayer;
```
