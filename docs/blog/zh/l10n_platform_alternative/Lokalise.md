# 用于替代 Lokalise 的开源 L10n 替代方案（TMS）

# 翻译管理系统 (TMS)

翻译管理系统 (TMS) 是一种软件平台，旨在自动化并简化翻译与本地化 (L10n) 流程。传统上，TMS 作为一个中心枢纽，用于上传和组织内容，并将其分配给人工译者。它负责管理工作流，存储翻译记忆（以避免重复翻译相同的句子），并处理将翻译后的文件交付回开发人员或内容管理员的过程。

本质上，TMS 历来是技术代码（字符串所在处）与理解文化的人工译者之间的桥梁。

# Lokalise

Lokalise 是现代 TMS 领域的重要参与者。其成立于 2017 年，通过高度关注开发者体验 (DX) 和设计集成颠覆了市场。与老牌竞争对手不同，Lokalise 优先打造流畅的 UI、强大的 API，以及与 Figma 和 GitHub 等工具的集成，以减少文件往返移动的摩擦。

它的成功建立在作为“开发者友好型” TMS 之上，能够自动化字符串的提取与插入，从而节省工程师的时间。它有效地解决了快速发展的技术团队希望摆脱手动处理电子表格邮件的需求，实现了**持续本地化**。

# Intlayer

Intlayer 主要作为 i18n 解决方案而闻名，但它也集成了一个无头 CMS。与主要作为外部字符串同步工具的 Lokalise 不同，Intlayer 更贴近您的代码。它控制整个技术栈——从打包层到远程内容交付——从而带来更顺畅、更高效的内容流。

## 为什么 AI 的出现改变了范式？

Lokalise 将本地化的 “DevOps” 方面做到了极致 —— 即自动化的字符串移动。然而，大型语言模型 (LLM) 的出现从根本上改变了本地化的范式。现在的瓶颈不再是**移动**字符串，而是**生成**内容。

随着 LLM 的出现，翻译成本急剧下降，速度呈指数级提升。本地化团队的角色正从“管理译者”转向“管理上下文和审核”。

尽管 Lokalise 已经增加了 AI 功能，但它本质上仍是一个为管理人工工作流而设计，并按席位或翻译键数量计费的平台。在 AI 优先的世界里，价值在于您如何编排 AI 模型以生成具有上下文感知的内容，而不仅仅是多容易地将任务分派给代理机构。

目前，最有效的工作流是先利用 AI 进行页面翻译和全球布局。然后，在第二阶段，当产品已经开始产生收入时，再聘请专业文案人员对特定的高流量内容进行优化，以提升转化率。

## 为什么 Intlayer 是 Lokalise 的理想替代方案？

Intlayer 是诞生于 AI 时代的解决方案。其架构基于这样一个原则：纯粹的翻译已成为一种商品，而**上下文**才是核心。

Lokalise 常因其陡峭的定价阶梯而受到诟病，这对于正在扩张的初创公司来说可能极其昂贵。Intlayer 则采用了不同的策略：

1.  **成本效率：** 您不会被锁定在按“翻译键数量”或“席位”计费的模型中，这种模型往往会惩罚公司的增长。使用 Intlayer，您可以根据自己的推理需求付费（自带 API 密钥，BYO Key），这意味着您的成本直接随实际使用量增长，而不是支付给平台的利润率。
2.  **工作流集成：** 虽然 Lokalise 需要同步文件（即使是自动化的），但 Intlayer 允许您在组件文件（React、Next.js 等）中直接通过**声明式内容**进行定义。这能将上下文保留在 UI 旁，减少错误。
3.  **可视化管理：** Intlayer 提供了一个可视化编辑器，可以直接与运行中的应用交互，确保在完整的视觉上下文中进行编辑 —— 而这在传统的 TMS 文件列表中往往是缺失的。

# 并排比较

| 功能         | Lokalise（现代 TMS）                           | Intlayer（AI 原生）                   |
| :----------- | :--------------------------------------------- | :------------------------------------ |
| **核心理念** | 自动化与设计阶段的本地化（L10n）。             | 管理内容逻辑与 AI 生成。              |
| **定价模式** | 按席位 / 月活（MAU）/ 键位数量计费（成本高）。 | 按自有推理付费（自带密钥，BYO Key）。 |
| **集成方式** | 基于 API 的同步 / Figma 插件。                 | 深度代码集成（声明式）。              |
| **更新**     | 同步延迟 / 需要创建 PR。                       | 与代码库或运行中应用即时同步。        |
| **文件格式** | 格式无关（移动、Web、文档）。                  | 现代 Web（JSON, JS, TS）。            |
| **测试**     | 审核工作流。                                   | CI / CLI / A/B 测试。                 |
| **托管**     | SaaS（闭源）。                                 | 开源 & 可自托管（Docker）。           |

Intlayer 提供了一个完整的一体化 i18n 解决方案，允许对内容进行深度集成。您的远程内容可以直接与代码库或在线应用同步。相比之下，Lokalise 通常依赖于创建拉取请求 (PR) 来更新代码仓库中的内容，这导致了“内容状态”和“应用状态”之间的分离。

此外，Intlayer 还可以作为功能开关 (Feature Flag) 或 A/B 测试工具使用，使您能够动态地测试不同的内容变体。Lokalise 专注于文本的准确性，而 Intlayer 则通过动态数据服务专注于优化**用户体验**。

Lokalise 非常适合移动应用（iOS/Android）和以设计为导向的工作流。然而，对于使用 Next.js 或 React 等框架的现代 Web 应用，Intlayer 对 `.js`、`.ts` 和 JSON 字典的原生处理提供了更优的开发者体验 (DX)，并为内容提供完整的 TypeScript 支持 —— 确保您再也不会部署缺失翻译键的代码。

最后，对于那些优先考虑数据主权与控制权的用户，Intlayer 是开源的且支持自托管。Docker 文件已直接包含在代码库中，赋予您对本地化基础设施的完全所有权 —— 这与 Lokalise 的封闭式 SaaS 模型形成了鲜明对比。
